## Learnings
- `df /` on macOS reports the sealed system volume only (~12G/229G free, misleading). Real user-data usage lives on `/System/Volumes/Data` — that's the number to lead with.
- HuggingFace model cache at `~/.cache/huggingface` (25G here) is a frequently-overlooked hog — worth checking on any ML-adjacent user's machine.
- Rust `target/` dirs explode when worktrees each carry their own (`target-diff-feature`, `target-ship-verify` siblings added 6G on top of the primary 10G).

## Conventions and decisions
- For disk audits, default to: `df /System/Volumes/Data` → ~/Library/{Caches,Application Support,Developer} → ~/projects → ~/.cache → /Applications → ~/Downloads, all in parallel `du -sh` passes. Cheap, comprehensive.
- Never delete user files unprompted. Present a sized, risk-labelled table and let the user pick batches.
